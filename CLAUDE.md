# CLAUDE.md

Claude Code가 이 저장소에서 작업할 때 반드시 지켜야 하는 규칙이다.
이 파일의 규칙은 사용자의 개별 프롬프트보다 우선한다. 충돌하면 작업을 멈추고 사용자에게 알린다.

## 프로젝트 개요

주간보고 관리 웹앱 (React 19 + TypeScript + Vite, localStorage 저장, 서버 없음).
이 저장소의 진짜 목적은 앱 자체가 아니라 [PROJECT.md](PROJECT.md)에 있다.
AI가 만든 코드를 "키울 수 있는 구조"로 바꾸는 학습 프로젝트다.

- 확장 담당: HY (기능을 추가한다, 커밋 접두어 `[HY/expand]`)
- 정리 담당: JK (구조를 다듬는다, 커밋 접두어 `[JK/refactor]`)
- 문서 작업: 접두어 `[docs]`
- 작업 순서: [docs/ROADMAP.md](docs/ROADMAP.md)의 현재 위치를 따른다

## 하드 규칙 (프로세스 보호, 절대 위반 금지)

1. **1 프롬프트 = 1 커밋.** 요청받은 작업 범위를 벗어난 파일을 건드리지 않는다.
   "하는 김에" 다른 코드를 정리하거나 개선하지 않는다.
   (커밋과 프롬프트의 1:1 대응이 깨지면 이 실험 전체가 무너진다)
2. **빌드 통과 전 커밋 금지.** 커밋 전 반드시 `npm run build`가 성공해야 한다.
3. **`[HY/expand]` 확장 작업 중에는 docs/review-notes.md 와 docs/jk-log.md 를
   읽거나 참고하지 않는다.** 이 파일들은 사람의 진단용 답안지다.
   확장 결과물이 답안지의 영향을 받으면 실험이 오염된다.
   (`[JK/refactor]` 정리 작업에서는 자유롭게 읽어도 된다)
4. ROADMAP.md의 현재 위치와 다른 작업을 요청받으면,
   순서가 어긋났음을 먼저 알리고 진행 여부를 확인한다.
5. 작업 완료 후: 사용된 프롬프트 원문을 docs/prompts/NNN-이름.md 로 박제하고
   (형식은 docs/prompts/_TEMPLATE.md), ROADMAP.md 체크박스와 현재 위치,
   PROJECT.md 진행 로그에 한 줄을 갱신한 뒤, 이것들을 같은 커밋에 포함한다.
6. 커밋 전 git status와 변경 요약을 사용자에게 보여주고 확인받은 후 커밋 앤 푸시한다.

## 이 파일에 없는 것 (의도적)

코드 품질, 설계 방식, 파일 구조에 대한 규칙은 의도적으로 두지 않는다.
확장 작업은 요청받은 기능을 AI 나름의 방식으로 구현하면 되고,
그 결과물을 진단하고 개선하는 것이 이 프로젝트의 학습 내용이다.

## 저장소 구조

```
CLAUDE.md            이 파일. 프로세스 규칙 (모든 세션이 읽음)
PROJECT.md           프로젝트 헌법. 목적, 학습 로드맵, 진행 로그 (진행 로그는 여기 하나만)
README.md            앱 사용법
docs/
  ROADMAP.md         작업 순서표와 현재 위치, 단계별 요청문 초안
  review-notes.md    사람 전용 답안지 (확장 작업 시 읽기 금지, 규칙 3)
  hy-log.md          HY의 확장 기록
  jk-log.md          JK의 진단과 정리 기록 (확장 작업 시 읽기 금지, 규칙 3)
  prompts/           실제 사용한 프롬프트 원문 박제 (번호순)
src/                 앱 코드
```
