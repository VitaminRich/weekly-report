# NNN: 작업 이름

- 작성자: HY 또는 JK
- 날짜: YYYY-MM-DD
- ROADMAP 항목: (예: 확장 1a)
- 커밋 메시지: `[HY/expand] 1a: ...`

## 프롬프트 원문

(Claude Code에 실제로 붙여넣은 내용을 그대로 보존한다. 아래는 프롬프트에 반드시
포함해야 하는 4요소다.)

```
[목적]
왜 이 작업을 하는지 한두 문장.

[작업 범위]
바꿀 파일과 내용을 구체적으로.

[하지 말 것]
- 이 작업과 무관한 파일 수정 금지
- (해당 시) 저장 형식 변경 금지 / 변경 시 마이그레이션 필수
- 스타일이나 구조 "개선" 임의로 하지 않기

[완료 기준]
- npm run build 통과
- (수동 테스트 시나리오를 구체적으로: 예 "기존 보고서가 그대로 열린다",
  "섹션을 추가하고 저장 후 새로고침해도 유지된다")
```
