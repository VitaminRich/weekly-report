## 운영 원칙 (환자 보호)

- 확장 프롬프트는 **사용자 말투**로 원하는 결과만 말한다.
  설계 방식, 파일 구조, 구현 단위를 지시하지 않는다. (000-baseline.md와 같은 톤)
- 검증은 프롬프트 밖에서 한다. 각 단계의 검증 체크리스트는 아래에 있고,
  결과는 프롬프트 박제 파일의 실행 결과 절에 기록한다.
- 확장 작업 전 보험: 기존 보고서를 HTML 다운로드로 백업해 둔다.
  (AI가 저장 형식을 깨서 데이터가 날아가면, 복구하고 사고 자체를 기록한다.
  그 사고가 블로그 소재다)
- 단계별 프롬프트 정밀도를 일부러 다르게 해보는 것도 실험 변수가 될 수 있다.
  시도했다면 hy-log.md에 기록한다.
